Módulo de Perfil e Inicio (Portal Médico)
Esta API alimenta la pantalla de inicio (home) del doctor al iniciar su turno.

    GET /v1/views/doctor/home

        Respuesta esperada:
            - doctor_nombre_pila
            - total_citas_hoy

            - proxima_cita: # (Puede ser null)
                - paciente_nombre
                - paciente_foto_url
                - motivo_consulta
                - fecha_relativa # ej. "Ahora mismo"
                - hora_formateada
                - ubicacion_consultorio
                - id_paciente # Para el botón "Ver Expediente"

            - quick_actions: (lista)
                - action_id
                - label
                - icon_id
                - route_name

Módulo de Agenda (Mi Agenda)
Esta es la sección para que el médico gestione sus citas del día y días posteriores.

    GET /v1/views/doctor/agenda_proximas

        Respuesta esperada:
            - cita_destacada: # La más próxima
                - id_cita
                - paciente_nombre
                - motivo_consulta
                - ubicacion_consultorio
                - fecha_hora_resumen # ej. "Hoy • 10:30 AM"
                - estado_badge # ej. "CONFIRMADA"
                
            - resto_del_dia: (lista)
                - id_cita
                - paciente_nombre
                - motivo_consulta
                - ubicacion_consultorio
                - hora_formateada
                
            - finalizadas_hoy: (lista)
                - id_cita
                - paciente_nombre
                - motivo_consulta
                - hora_formateada
                - estado_badge # ej. "COMPLETADA"

Módulo Mis Pacientes (Directorio y Búsqueda)
Pantalla principal para buscar y ver el listado de pacientes atendidos.

    GET /v1/views/doctor/patients

        Respuesta esperada:
            - pacientes_recientes: (lista) # Puede cambiar a 'todos' según el filtro
                - id_paciente
                - foto_perfil_url
                - nombre_completo
                - id_paciente_corto # ej. "ID: 88210"
                - indicador_estado # Booleano para el punto verde de "Cita Hoy"
                - cita_info_resumen # ej. "Próxima cita: Hoy, 4:00 PM" o "Última consulta: 15 Oct"


Módulo Expediente Médico (Historial por Paciente)
Vista del historial de consultas previas de un paciente específico.

    GET /v1/views/doctor/patients/{id_paciente}/history

        Respuesta esperada:
            # Datos del Paciente
            - paciente_nombre
            - paciente_foto_url
            
            # Línea de tiempo
            - historial_consultas: (lista)
                - id_consulta
                - fecha_completa # ej. "15 DE AGOSTO, 2023"
                - diagnostico_titulo # ej. "Seguimiento de Hipertensión"
                - resumen_notas
                - enlace_resumen_disponible # Booleano para mostrar "Ver resumen médico"

Módulo Expediente Clínico (Nueva Consulta)
Datos necesarios para cargar la cabecera antes de llenar las notas y para guardar la consulta.

    GET /v1/views/doctor/consultations/new?id_paciente={id}

        Respuesta esperada (Datos de cabecera pre-cargados):
            - paciente_nombre
            - paciente_foto_url
            - id_paciente_formateado # ej. "PACIENTE ID: AS-2024-098"
            - edad_anios
            - peso_kg
            - alergias_resumen # Texto en rojo, ej. "Penicilina"
            
    Acción: Guardar y Firmar Expediente Clínico
        Se utiliza al finalizar la consulta, presionando "Firmar y Guardar Expediente".

        POST /v1/doctor/consultations

            Cuerpo de la petición (Lo que se envía):
                - id_paciente
                - id_cita
                - notas_interrogatorio
                - notas_exploracion_fisica
                - notas_diagnostico
                - notas_tratamiento_receta
                - archivos_adjuntos_urls: (lista de URLs o Base64)
                - consentimiento_informado_firmado # Booleano (Checkbox)
                - firma_medico_base64 # Imagen del canvas de firma digital

            Respuesta esperada:
                - id_expediente_generado
                - mensaje # ej. "Expediente guardado y firmado correctamente"

Módulo Historial de Pagos (Finanzas del Médico)
Vista de los ingresos generados y transacciones recientes.

    GET /v1/views/doctor/payments

    Respuesta esperada:
        - ingresos_del_mes_formateado # ej. "$45,800.00"
        
        - transacciones_recientes: (lista)
            - id_transaccion
            - paciente_nombre
            - concepto_resumen # ej. "Consulta - Ananna Smith"
            - fecha_corta # ej. "24 Oct"
            - metodo_pago_resumen # ej. "PAGO CON TARJETA" o "ANTICIPO"
            - monto_formateado
            - estado_pago # ej. "LIQUIDADO" o "PAGO PENDIENTE"

    Acción: Exportar Historial
        GET /v1/doctor/payments/export?mes={YYYY-MM}

            Respuesta esperada: 
                URL de descarga del archivo PDF/CSV o un stream de archivo.

Módulo Mi Perfil (Doctor)
Vista de los datos profesionales y de contacto del médico.

    GET /v1/views/doctor/profile_details

        Respuesta esperada:
            - foto_perfil_url
            - nombre_completo # ej. "Dr. James Wilson"
            - nivel_especialista # ej. "ESPECIALISTA SENIOR"

            # Información Profesional
            - especialidad_nombre
            - experiencia_anios

            # Información Personal
            - nombre_legal_completo
            - cedula_profesional
            - fecha_nacimiento

            # Contacto
            - telefono
            - correo_electronico

Módulo de Acciones Rápidas (Mutaciones Extras)

Acción: Bloquear Horario (Ajustes de disponibilidad)
Se utiliza para inhabilitar horas o días enteros en la agenda del doctor.

    POST /v1/doctor/schedule/blocks

        Cuerpo de la petición (Lo que se envía):
            - id_ubicacion # ej. Ambas, Las Américas, La Primavera
            - fechas_seleccionadas: (lista de fechas ISO)
            - bloquear_todo_el_dia # Booleano
            - hora_inicio
            - hora_fin
            - motivo_bloqueo # ej. "Asunto personal"
            - forzar_bloqueo_con_reembolso # Booleano (Para sobreescribir advertencias)

        Respuesta esperada (Éxito):
            - mensaje # "Horario bloqueado correctamente"
            - citas_canceladas_count
            
        Respuesta esperada (Error 409 - Conflicto detectado, si `forzar_bloqueo_con_reembolso` es false):
            - codigo_error: "CONFLICTO_AGENDA"
            - mensaje_advertencia: "Tienes 1 cita programada en este horario..."
