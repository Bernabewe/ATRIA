Módulo de Perfil y Autenticación
    Esta API alimenta la pantalla de inicio (home) del usuario paciente.

    GET /v1/views/patient/home

    Respuesta esperada:
        - nombre_pila
        - foto_perfil_url
        - estado_salud_resumen

        - next_cita: # (Puede ser null si no hay citas)
            - id_cita
            - doctor_nombre
            - especialidad_etiqueta
            - fecha_relativa # "Mañana", "Hoy"
            - hora_formateada
            - ubicacion_resumen
            - badge_status

        - quick_actions:
            - action_id
            - label
            - icon_id
            - route_name
            - is_enabled

Módulo de Citas

    GET /v1/views/patient/citas_proximas

    Respuesta esperada:

        - proxima_prioridad:
            - id_cita
            - doctor_nombre
            - doctor_foto_url
            - especialidad_etiqueta       
            - estado_badge              
            - fecha_hora_resumen      

        - visitas_proximas: (lista)
            - id_cita
            - doctor_nombre
            - motivo_consulta
            - fecha_corta
            - icono_especialidad_id

        - completadas_recientemente: (lista)
            - id_cita
            - doctor_nombre
            - motivo_consulta
            - fecha_corta
            - icono_especialidad_id
            - estado_texto
            - estado_icono
    Pasadas: Esta es la sección para que el paciente vea el historial de citas pasadas.

    GET /v1/views/patient/citas_pasadas

    Respuesta esperada:
        - historial_citas: (lista)
            - id_cita
            - doctor_nombre
            - motivo_consulta
            - fecha_corta
            - icono_especialidad_id
            - estado_texto
            - estado_icono

Módulo Detalles Citas
    Esta es la sección para que el paciente gestione los detalles de la cita seleccionada.

    GET /v1/views/patient/gestionar_cita?id_cita={id}

    Respuesta esperada:

        # Datos del Médico
        - doctor_nombre                   
        - doctor_foto_url                            
        - especialidad_titulo      

        # Resumen de la Consulta               
        - estado_badge                     
        - fecha_completa                  
        - hora_formateada                 
        - ubicacion_completa              

        # Detalles de Cobro
        - anticipo_monto

        # Políticas y Alertas
        - alerta_cancelacion: # (Puede ser null)             
            - titulo
            - mensaje

        # Acciones disponibles 
        - permitir_cancelacion

Módulo Reserva Médica (Flujo de Agendamiento)
    Paso 1: Selección de Especialidad

        GET /v1/views/patient/reservas/especialidad

        Respuesta esperada:
            - especialidades: (lista)
                - id_especialidad
                - nombre_especialidad
                - descripcion_corta
                - icono_id
            
    Paso 2: Selección de Sucursal

        GET /v1/views/patient/reservas/sucursal

        Respuesta esperada:
            - sucursales: (lista)
                - id_sucursal
                - nombre_sucursal
                - direccion_completa
                - coordenadas:
                    - latitud
                    - longitud

    Paso 3: Selección de Especialista

        GET /v1/views/patient/reservas/doctores?id_especialidad={id}&id_sucursal={id}

        Respuesta esperada:
            - especialistas: (lista)
                - id_doctor
                - nombre_doctor
                - foto_url
                - calificacion_promedio
                - experiencia_anios
                - especialidad_titulo

    Paso 4: Selección de Fecha y Hora

        GET /v1/views/patient/reservas/disponibilidad?id_doctor={id}&mes={YYYY-MM}

        Respuesta esperada:
            - dias_disponibles: (lista de fechas)
            - horarios_por_dia: (lista)
                - hora_formateada
                - estado # "disponible" o "ocupado"

    Paso 5: Resumen Final antes del Pago

        GET /v1/views/patient/reservas/resumen?id_doctor={id}&id_sucursal={id}&fecha={date}&hora={time}

        Respuesta esperada:
            - doctor_nombre
            - doctor_foto_url
            - especialidad_titulo
            - sucursal_nombre
            - sucursal_direccion
            - fecha_hora_formateada
            - duracion_estimada
            - costo_total
            - anticipo_requerido

Módulo Historial de Pagos
    Esta es la sección para el control financiero y pagos pendientes.

    Respuesta esperada:

Módulo de Acciones (Actualización y Creación)
    Esta sección contiene los endpoints que el paciente utiliza para interactuar con el sistema, modificar sus datos o realizar transacciones.

    Acción: Editar Perfil
    Se utiliza al presionar el botón "Editar Perfil" para actualizar la información personal.

    PUT /v1/patient/profile

        Cuerpo de la petición (Lo que se envía):
            - peso
            - altura
            - telefono
            - direccion_completa

        Respuesta esperada:
            - mensaje # ej. "Perfil guardado con éxito"


    Acción: Pagar Anticipo / Crear Cita (Paso 5)
    Se utiliza al presionar el botón "Pagar Anticipo" para finalizar el flujo de agendamiento.

    POST /v1/reservas/appointments

        Cuerpo de la petición (Lo que se envía):
            - id_doctor
            - id_sucursal
            - fecha_hora_inicio
            - metodo_pago_id # ID de la tarjeta o token de pago

        Respuesta esperada:
            - id_cita
            - estado_pago # ej. "APROBADO"


    Acción: Pagar Pendiente (Historial de Pagos)
    Se utiliza al presionar el botón "Pagar Ahora" en un adeudo registrado.

    POST /v1/patient/payments/{id_transaccion}/pay

        Cuerpo de la petición (Lo que se envía):
            - metodo_pago_id

        Respuesta esperada:
            - nuevo_estado # ej. "PAGADO"
            - fecha_pago


    Acción: Cancelar Cita (Detalles de Cita)
    Se utiliza al confirmar la alerta de "Cancelar Cita" para cambiar el estado de la reserva.

    PATCH /v1/appointments/{id_cita}/cancel

        Cuerpo de la petición (Lo que se envía):
            - motivo_cancelacion # ej. "Problemas de transporte"

        Respuesta esperada:
            - nuevo_estado # ej. "CANCELADA"
            - reembolso_aplicado # booleano
